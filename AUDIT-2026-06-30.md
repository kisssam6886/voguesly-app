## 🔬 native 隐私 wave 调研结论（2026-07-01，实地查 HK 后端 default.clash.yaml）

**重大发现:之前审计标 P0/P1 嘅隐私漏洞,大部分系误报——你嘅订阅模板已经做好咗。**

- **DNS 泄漏 = 唔成立** ✅：模板有正经 `dns` block(`enhanced-mode: fake-ip` + DoH doh.pub/alidns/onedns + fallback-filter geoip)。app 嘅 `system://` fallback 只喺配置无 dns 时触发,而模板 `dns.enable:true` → **唔触发,无泄漏**。
- **IPv6 泄漏 = 低残留**：`dns.ipv6:false` 已令域名只解析 IPv4(绝大多数流量入隧道);只剩字面 IPv6/IPv6-only 站边缘风险。低优先,可选(set 顶层 ipv6:true 拦截,需设备测)。
- **节点「未选择·去选线路」= 纯显示问题**:主组 `select` 默认走第一项「🚀快线·通用」(url-test 自动选最快)→ **连接其实 work**,唔系「连了个寂寞」。只需改显示(显「默认·快线」),唔使改选路。

### ⚠️ 唯一真 P0 残留:VPN 授权被拒仍显「已连接」(流量裸奔)
**根因(精确定位)**：
- native `AppPlugin.kt:442 onActivityResult` 只 handle `RESULT_OK` → 用户㩒「拒绝」VPN 授权时**无 else,静默 drop**;`vpnPrepareCallback` 永不 fire → `Service.startService` 永不行 → `runStateFlow` 卡 PENDING。
- Dart `action.dart:197 updateStatus(init)` **预先** set `runTimeProvider=0`(=isStart=true=「已连接」);`_handleStart` 只 kick off prepare 唔 await 真正建立,deny 时唔抛异常 → `catch`(line 205-206)唔触发 → runTime 卡 0 → 永显已连接。

**修法(跨 native + Dart,需设备测)**：
1. native onActivityResult 加 `else`:deny 时 `vpnPrepareCallback=null` + `runStateFlow.tryEmit(RunState.STOP)`。
2. Dart 侧观察 deny/STOP → 回滚 `runTimeProvider=null` + `globalState.showNotifier('未授权 VPN,连接失败')`。

**⚠️ 唔盲 ship**：呢个改跨 Kotlin 核心 + Dart method-channel,改错会令所有人连接 wedge,而 headless 环境冇法 device-test。建议你设备 in-the-loop 时一齐做+测(㩒拒绝授权验证显错)。
